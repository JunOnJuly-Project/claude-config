# 프로젝트 계획

사용자의 요구사항을 분석하고 구조화된 개발 계획을 수립합니다.

## 입력
`$ARGUMENTS` — 계획할 프로젝트 또는 기능 설명

## 실행 단계

### 1. 프로젝트 환경 분석
- Glob으로 프로젝트 구조 파악 (package.json, pyproject.toml, tsconfig.json 등)
- 기존 코드 패턴, 프레임워크, 디렉토리 구조 확인
- 프로젝트가 없으면 새 프로젝트로 판단

### 2. 요구사항 분석
- `$ARGUMENTS`에서 핵심 기능 요구사항 추출
- 기능적/비기능적 요구사항 분리
- 불명확한 사항은 사용자에게 질문

### 3. 도메인 식별
- 핵심 도메인과 보조 도메인 구분
- 도메인 간 의존성 관계 매핑
- 도메인별 브랜치 네이밍 제안: `feature/{도메인}/{기능}`

### 4. 작업 분해
도메인별로 최소 구현 단위로 태스크 분해:
- 각 태스크에 우선순위 (높음/중간/낮음) 부여
- 의존성 순서 표시
- 예상 커밋 수 산정

### 5. 계획 문서 생성
`docs/plans/` 디렉토리에 아래 형식으로 문서 생성:

```markdown
# {프로젝트/기능} 개발 계획
- 작성일: {YYYY-MM-DD}

## 개요
{요구사항 요약}

## 도메인 분석
{식별된 도메인, 관계도}

## 디렉토리 구조 (제안)
{트리 형식 구조}

## 작업 분해
### 도메인: {이름}
- [ ] {태스크} — 우선순위: {수준}, 브랜치: feature/{도메인}/{기능}
  - 커밋 계획: {예상 커밋 목록}

## 기술 결정사항
{선택 기술과 근거}

## GitHub 이슈 계획
{생성할 이슈 목록과 라벨}

## 위험 요소
{잠재 문제와 대응 방안}
```

## 출력
- 모든 내용은 **한국어**로 작성
- 계획 문서 파일 경로 출력
- 사용자에게 계획 요약 보고
