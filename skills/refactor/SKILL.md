# 코드 리팩토링

코드의 외부 동작을 변경하지 않으면서 내부 구조를 개선합니다.

## 입력
`$ARGUMENTS` ��� 리팩토링 대상 (파일, 모듈, 또는 리팩토링 유형)

## 실행 단계

### 1. 현황 분석
- 대상 코드의 현재 구조 파악
- 문제점 식별:
  - SOLID 위반 사항
  - DRY 위반 (중복 코드)
  - 과도한 복잡도 (긴 함수, 깊은 중첩)
  - 부적절한 네이밍
  - 매직넘버/문자열

### 2. 테스트 확보
- 기존 테스트 존재 여부 확인
- 테스트 부족 시 **리팩토링 전** 테스트 먼저 작성
- 리팩토링 안전망 확보

### 3. 리팩토링 실행
지원하는 리팩토링 기법:
- **Extract Method**: 긴 함수에서 의미 단위 추출
- **Extract Class**: 과도한 책임을 가진 클래스 분리
- **Move to Layer**: 잘못된 레이어의 코드 이동
- **Introduce Interface**: 구체 의존을 추상 의존으로 전환
- **Replace Conditional with Polymorphism**: 복잡한 조건문을 다형성으로
- **Introduce Value Object**: 원시값을 의미있는 객체로
- **Introduce Constant**: 매직넘버를 상수로 추출
- **Rename**: 의미를 명확하게 하는 이름 변경

### 4. 점진적 적용
- 한 번에 하나의 리팩토링 기법만 적용
- 각 리팩토링 후 테스트 실행
- 테스트 통과 확인 후 다음 리팩토링
- 각 단계마다 개별 커밋

### 5. 커밋
- `refactor({도메인}): {리팩토링 설명}`
- 본문에 적용한 기법과 이유 기술

## 출력
```markdown
## 리팩토링 보고서
- 대상: {파일/모듈}
- 적용한 기법: {목록}
- 변경 전/후 비교: {구조적 개선 사항}
- 테스트 결과: 전체 통과 확인
- 커밋 히스토리: {커밋 목록}
```

## 주의
- 동작 변경 금지 — 리팩토링은 구조만 변경
- 테스트 없이 리팩토링 금지
- 모든 출력 **한국어**
