# /review — 코드 리뷰

## 호출 형식
```
/review [--strict] [{파일/디렉터리 또는 PR URL}]
```

## 파이프라인
1. 대상 자동 감지: 인자 없으면 `git diff` (unstaged + staged)
2. **reviewer** 에이전트 호출 — 읽기 전용
3. 리포트 출력: 심각도 마커별 분류
4. (선택) `--fix` 플래그 있으면 developer 로 자동 수정 위임

## 심각도 마커
- `[치명적]` — 보안/데이터 손실/크래시. 즉시 수정
- `[중요]` — 로직 오류/성능. 머지 전 수정
- `[제안]` — 네이밍/구조 개선
- `[칭찬]` — 잘 작성된 부분 인정

## --strict
`[중요]` 1개만 있어도 차단 (기본은 경고 통과).
