# 코드 리뷰

클린코드 원칙에 기반한 체계적 코드 리뷰를 수행합니다.

## 입력
`$ARGUMENTS` — 리뷰 대상 (파일 경로, PR 번호, 또는 비어있으면 git diff 대상)

## 실행 단계

### 1. 리뷰 범위 결정
- `$ARGUMENTS`에 파일 경로 지정 시: 해당 파일 리뷰
- PR 번호 지정 시: `gh pr diff` 분석
- 비어있으면: `git diff` (스테이징 + 언스테이징 변경사항)

### 2. 체크리스트 기반 리뷰

**코드 품질**:
- 함수 20줄 이내, 단일 책임
- 네이밍 규칙 (동사/명사/불리언 접두사)
- 매직넘버 없음, 상수 사용
- 한국어 주석 (WHY만)
- DRY/KISS 준수

**SOLID 원칙**:
- SRP, OCP, LSP, ISP, DIP 위반 여부

**보안**:
- SQL 인젝션, XSS, 입력값 검증
- 민감 정보 하드코딩, 인증/인가

**성능**:
- N+1 쿼리, 불필요한 반복, 메모리 누수

**테스트**:
- 테스트 커버리지, 엣지 케이스 포함 여부
