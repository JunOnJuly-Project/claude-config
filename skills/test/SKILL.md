# 테스트 생성 및 실행

대상 코드에 대한 테스트를 생성하고 실행합니다.

## 입력
`$ARGUMENTS` — 테스트 대상 (파일, 모듈, 함수명, 또는 비어있으면 전체)

## 실행 단계

### 1. 환경 감지
- 테스트 프레임워크 자동 감지:
  - JavaScript/TypeScript: jest, vitest, mocha (package.json 확인)
  - Python: pytest, unittest (pyproject.toml, setup.cfg 확인)
  - Go: 내장 testing
  - Java/Kotlin: JUnit, Kotest
- 기존 테스트 패턴 분석 (테스트 디렉토리, 네이밍 규칙)

### 2. 대상 분석
- 대상 함수/클래스의 시그니처, 의존성 파악
- 입력 타입, 반환 타입, 예외 가능성 분석
- 외부 의존성 식별 (Mock 필요 여부 판단)

### 3. 테스트 작성
각 대상에 대해:
- **정상 경로**: 기대 동작 검증
- **엣지 케이스**: 경계값, 빈 값, null
- **에러 조건**: 잘못된 입력, 예외
- **경계값**: 최소/최대, 0, 음수

테스트명은 한국어:
```javascript
it('유효한 이메일로 로그인하면 토큰을 반환한다', () => {});
```
```python
def test_유효한_이메일로_로그인하면_토큰을_반환한다(self):
```

### 4. 실행
- 테스트 실행 및 결과 확인
- 실패 시 원인 분석 (테스트 코드 vs 프로덕션 코드 문제 구분)
- 가능한 경우 커버리지 측정

### 5. 보고
```markdown
## 테스트 결과
- 총 테스트: {n}개
- 통과: {n}개 | 실패: {n}개 | 스킵: {n}개
- 커버리지: {n}% (측정 가능 시)

### 실패 테스트 상세
- {테스트명}: {실패 원인}
```

## 커밋
- `test({도메인}): {테스트 설명}` 형식으로 커밋
